# 错误处理指南

> 版本: 3.2.0 | 最后更新: 2026-02-05

本文档说明常见错误及处理方法。

---

## 错误类型一览

| 错误代码 | 含义 | 处理方式 |
|----------|------|----------|
| `path_not_found` | 路径不存在 | 提示用户检查路径 |
| `no_nc_files` | 目录下无 NC 文件 | 提示用户确认目录 |
| `token_invalid` | 确认令牌无效 | 重新从阶段 1 开始 |
| `token_missing` | 缺少确认令牌 | 补充 confirmation_token |
| `variable_not_found` | 变量不存在 | 展示可用变量让用户选择 |
| `shape_mismatch` | 形状不匹配 | 检查变量维度 |
| `permission_denied` | 权限不足 | 提示用户检查目录权限 |

## 路径相关错误

### path_not_found

**错误信息**：`数据目录不存在: /path/to/data`

**原因**：用户提供的路径不存在或拼写错误。

**处理**：
```
您提供的路径不存在：/path/to/data
请检查路径是否正确，或提供新的路径。
```

### no_nc_files

**错误信息**：`目录下没有找到 NC 文件`

**原因**：目录存在但没有 .nc 文件。

**处理**：
```
在目录 /path/to/data 下没有找到 NC 文件。
请确认：
1. 文件扩展名是否为 .nc
2. 是否需要指定子目录
```

## Token 相关错误

### token_invalid

**错误信息**：`确认令牌无效或已过期`

**原因**：
1. Token 已过期（超过 30 分钟）
2. 参数已变更导致 Token 失效
3. Token 被篡改

**处理**：
```
确认令牌已失效，需要重新开始流程。
我将重新分析您的数据...

[重新调用 ocean_sr_preprocess_full，不带 confirmation_token]
```

### token_missing

**错误信息**：`阶段 X 需要 confirmation_token`

**原因**：调用后续阶段时未提供 Token。

**处理**：从上一次返回中获取 `confirmation_token` 并传入。

## 变量相关错误

### variable_not_found

**错误信息**：`变量 xxx 在数据中不存在`

**原因**：用户指定的变量名在 NC 文件中找不到。

**处理**：
```
变量 "xxx" 在数据中不存在。

可用的动态变量：
- chl: 叶绿素
- no3: 硝酸盐
- temp: 温度

请从以上列表中选择。
```

### shape_mismatch

**错误信息**：`变量形状不一致`

**原因**：多个变量的空间维度不匹配。

**处理**：
```
检测到变量形状不一致：
- chl: (365, 680, 1440)
- temp: (365, 50, 680, 1440)

temp 是 4D 变量（包含深度维度），需要单独处理。
请选择形状一致的变量进行处理。
```

## 系统错误

### permission_denied

**错误信息**：`无法写入目录: /path/to/output`

**处理**：
```
无法写入输出目录，请检查：
1. 目录是否存在
2. 是否有写入权限
3. 磁盘空间是否充足
```

### python_error

**错误信息**：`Python 脚本执行失败`

**处理**：
1. 检查 Python 环境
2. 查看详细错误信息
3. 如果是依赖问题，提示安装

## 错误恢复流程

当遇到错误时，Agent 应该：

1. **停止执行**：不要尝试自动修复
2. **展示错误**：清楚说明什么出错了
3. **给出建议**：提供可能的解决方案
4. **等待指示**：让用户决定下一步

**示例**：
```
执行过程中遇到错误：

【错误类型】变量不存在
【错误详情】变量 "chlorophyll" 在数据中不存在
【可能原因】变量名拼写错误

【建议操作】
1. 查看实际变量名：我检测到类似变量 "chl"
2. 重新选择变量

请问您想如何处理？
```
