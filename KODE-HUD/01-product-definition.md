# KODE-HUD 产品定义

## 1. 产品一句话定义

KODE-HUD 是 Ocean-Agent-SDK 的实时可视化监控台，用来持续展示 Agent 当前在做什么、作用在什么对象上、以及刚刚产出了什么。

## 2. 这个产品不是什么

它不只是：

- 一个聊天窗口
- 一个日志终端搬运器
- 一个只在任务结束后显示结果的结果页

它更像是：

- Agent 指挥看板
- Agent 运行态监控面板
- 多角色协作任务的实时观察台

## 3. 用户真正想解决的问题

用户最关心的不是“最终答案出来没”，而是：

- Agent 现在在做什么
- Agent 卡住了吗
- Agent 正在读哪个文件
- Agent 正在写哪个文件
- Agent 在运行什么命令或脚本
- Agent 最近产出了什么内容
- 多个 Agent 中，谁在工作，谁在等待，谁失败了

## 4. 核心用户问题

KODE-HUD 必须实时回答下面这些问题：

1. 现在有哪些 Agent 正在工作？
2. 每个 Agent 当前处于什么状态？
3. 每个 Agent 正在做什么动作？
4. 这个动作作用在什么对象上？
5. 最近 10 秒发生了什么？
6. 是否有 Agent 报错、超时、无心跳、卡死？
7. 这个 Agent 已经产出了哪些文件或结果？

## 5. 核心价值

### 5.1 可观测性

让 Agent 的内部执行过程从黑箱变成可见。

### 5.2 可控性

用户不必等任务结束才知道是否偏航，可以在中途发现问题。

### 5.3 可审计性

用户能回看某个 Agent 什么时候读了哪个文件、跑了哪个命令、生成了哪些产物。

### 5.4 多角色协作感知

用户能看到 Planner、Researcher、Coder、Reviewer 等角色是否真的在协作，而不是只看到最后一句总回复。

## 6. 第一优先级信息

根据当前需求，KODE-HUD 的第一优先级不是漂亮动画，而是下面两类信息：

### 6.1 当前动作

例如：

- 正在思考
- 正在调用工具
- 正在读取文件
- 正在写入文件
- 正在运行命令
- 正在执行训练
- 正在等待工具返回

### 6.2 当前对象

例如：

- `src/config.ts`
- `train.log`
- `main_ddp.py`
- `/data/WaveCastNet/processdata`
- `/outputs/report.md`

也就是：

`当前动作 + 当前对象`

这是 HUD 的核心组合。

## 7. MVP 定义

### MVP 必须做到

- 实时显示 Agent 列表
- 实时显示单个 Agent 当前状态
- 实时显示当前动作
- 实时显示当前目标对象
- 实时显示最近工具调用
- 实时显示最新文本输出
- 实时显示最近产出文件

### MVP 可以暂时不做

- 复杂 3D 动画
- AI 自动总结每个 Agent 的长周期历史
- 多租户权限体系
- 完整任务编排系统

## 8. 成功标准

如果用户打开页面后，能在 3 秒内回答下面这些问题，就算 HUD 方向是对的：

- 现在哪个 Agent 在跑？
- 它在干什么？
- 它碰的是哪个文件或脚本？
- 它最近产生了什么？
- 它是继续运行、卡住还是失败？

## 9. 与现有 Ocean-Agent-SDK 的关系

KODE-HUD 不是替代现有 Agent 服务，而是作为其上层观察面。

Ocean-Agent-SDK 负责：

- Agent 生命周期
- SSE 流式输出
- 工具调用
- 会话持久化

KODE-HUD 负责：

- 运行态可视化
- 多 Agent 看板
- 实时监控
- 事件聚合和状态投影
